# 분석 단계 지침 (daily-post 분석 단계)

이 단계는 초안 작성 전에 "투자 관점" 근거를 마련하는 단계다. 결과(분석 노트)는 파일로 저장하지 않고, 다음 단계(draft.md)에 바로 이어서 쓴다.

## 1. 세 렌즈로 분류 + 선행/동행 태그
이번 뉴스가 다음 중 무엇을 건드리는지 정한다(복수 가능):
1. **경기 흐름**(성장이 살아나나 식나)
2. **돈의 흐름/유동성**(금리·신용 여건이 풀리나 조이나)
3. **기업 체력**(실적·밸류에이션)

관련 지표가 있으면 아래 선행/동행 표로 시계를 분류한다. 표에 없는 지표를 다룰 때는 임의로 "동행"으로 간주하지 말고 본문에서 판단 근거를 밝힌다.

| 지표 | 시계 |
|---|---|
| 10Y–3M 스프레드 | 강선행(12~18개월) |
| Conference Board LEI | 선행 |
| ISM 제조업 PMI | 동행 |
| 미시간 소비자심리 기대-현황 격차 | 선행 |
| HY OAS 스프레드 | 동행 |
| Chicago Fed NFCI | 동행 |
| 10Y 실질금리(TIPS) | 동행 |
| S&P500 fwd EPS 성장률·리비전 | 선행 |

선행 지표가 경고 신호를 보내도 동행 지표가 아직 반영하지 않았다면, 최종 서술에서 "지금 당장"과 "앞으로의 경고"를 분리해 말한다. 이 구분이 분석 노트의 "선행 vs 동행" 필드다.

## 2. 관련 챕터만 골라 매크로 참고 노트 확인
`macro-reference.md`(같은 디렉터리)를 Read로 읽는다. 뉴스 주제와 맞는 챕터 절만 골라 참고하고, 다른 챕터는 읽지 않는다. 이 파일 자체가 원문 대체 요약이므로 원문 PDF는 더 이상 참조하지 않는다.

macro-reference.md도 "이 책에 의하면" 식으로 직접 인용하지 않는다. 인과 논리만 가져와 분석에 녹여쓴다.

## 3. 핵심 지표 1~2개만 타깃 검색 + 임계값 판정
이번 뉴스와 가장 직결된 지표 1~2개를 정해 WebSearch/WebFetch로 확인한다. 4모듈 전체 스냅샷을 재구축하지 않는다 — 그건 이 블로그의 역할이 아니다.

검색해 확인한 값은 아래 표로 🟢(양호)/🟡(중립)/🔴(경고) 중 하나로 판정한다. 표에 없는 지표는 임의로 판정하지 않고 "참고용"으로만 쓴다.

| 지표 | 🟢 | 🟡 | 🔴 |
|---|---|---|---|
| 10Y–3M 스프레드 | > +0.50% | 0 ~ +0.50% | < 0(역전) |
| LEI 6개월 연율 | > 0% | −3% ~ 0% | < −3% |
| ISM PMI | > 52 | 48 ~ 52 | < 48 |
| 소비자심리 기대-현황 격차 | 격차 축소·양호 | 중립 | 기대 급락 |
| HY OAS | < 350bp | 350~500bp | > 500bp |
| NFCI | < 0(완화) | −0.1~+0.1 | > 0(긴축) |
| 10Y 실질금리 | 하락·안정 | 횡보 | 급등(추세적 상승) |
| fwd EPS 성장률 | > +10%·리비전↑ | 0~+10% | 음(−)·리비전↓ |

- 검색 없이 지표 수치를 단정하지 않는다.
- 검색에 실패하면 "확인 불가"로 남긴다.
- 신선도: 일간 지표(금리·스프레드)는 영업일 3일, 월간 지표(PMI·심리)는 발표 주기 1회, 분기 지표(EPS)는 1분기를 넘기면 "기준일 경과 — 참고용"으로 표기한다.
- Fed 정책 관련 단정 표현 금지 규칙은 `writing-styles.md`의 "숫자·사실관계 규칙"을 따른다(이 문서에서 중복 규정하지 않는다).
