# 글쓰기 스타일 가이드 (daily-post 공통)

`.claude/daily-post/draft.md`가 포스트 초안(및 용어사전 초안)을 쓸 때 참조하는 문체·표현·수치 규칙 모음이다. 이 파일 자체는 독립 실행 단계가 아니라, draft.md의 각 절에서 "이 규칙을 따르라"고 가리키는 참조 대상이다.

## 문체·표현 규칙
- 마크다운 볼드(`**`)는 절대 쓰지 않는다. 강조는 어휘 선택이나 문장 구조로 표현한다. 포스트 본문과 용어사전 설명 모두에 적용된다.
- 비전공자가 이해할 난이도로 쓴다. 일상적인 비유를 적극 활용한다(예: 금리를 "이자 온도조절기"에 빗대기).
- 어려운 용어는 처음 등장하는 자리에서 반드시 짧게 풀어 쓰거나 일상적 비유를 붙인다(예: "PER" → "이익 대비 몸값을 매기는 저울"). 용어사전에 링크만 걸고 본문에서 안 풀어 쓰는 것은 이 규칙을 지킨 것으로 치지 않는다.
- 두괄식: 첫 문단(1~2문장)에서 "무슨 일이 있었나"에 곧장 답한다. "~에 대해 알아보겠습니다", "~를 짚어보겠습니다", "~가 화제입니다" 같은 예고형 서론으로 시작하지 않는다 — 검색해서 들어온 독자는 0.5초 안에 "여기 내 답이 있나"를 판단하고, 없으면 바로 이탈한다.
- 서론 직후, 인용블록(`>`)으로 핵심 요약을 2~3문장(무슨 일 + 왜 중요한지)으로 넣는다.
- 문체는 기존 발행글과 같은 존댓말체(~습니다/~입니다)로 통일한다. 뉴스 기사체 종결어미('~다/~했다/~이다')를 섞지 않는다.
- 문장 길이에 리듬을 준다 — 대부분 40~60자로 비슷하면 짧은 문장과 긴 문장을 섞는다.

## description 작성 규칙
- 100자 내외. 이 글을 읽으면 무엇을 알게 되는지 한 문장으로 압축한다.
- "~를 정리합니다", "~에 대해 알아봅니다" 같은 무미건조한 예고형 문장 금지 — 검색 결과·공유 미리보기에 그대로 노출되는 문장이므로, 클릭할 이유가 담겨야 한다.

## 표 사용 규칙
분석 노트의 "확인된 수치"가 2개 이상이면 본문 안에 마크다운 표로 정리한다(열: 지표 | 값 | 출처 | 기준일). 1개뿐이면 표 없이 문장으로 녹여 쓴다 — 표를 채우려고 억지로 항목을 늘리지 않는다.

## 숫자·사실관계 규칙
- 사실관계 검증: 숫자뿐 아니라 모든 사실관계(사건, 원인, 영향 등)에 대해 기사 원문이나 분석 노트의 확인된 자료에만 근거한다. 확인 불가한 사건·정황은 아예 언급하지 않고, 숫자처럼 독자가 구체값을 기대하는 항목만 "확인 불가"로 표시한다.
- 숫자 레드라인: 금리·스프레드·PER 등 구체 수치는 지어내지 않는다. 분석 노트에 확인된 값만 쓰고, 없으면 "확인 불가"로 남긴다. "Fed가 다음 달 내린다" 식 단정 예측 금지 — 필요하면 "시장은 인하 가능성에 무게를 두는 분위기" 정도로만(확인된 범위에서).
- 기준일 병기: 확인된 수치를 쓸 때는 항상 기준일을 함께 적는다(예: "2026년 7월 기준", "발표 시점 기준"). 기준일이 없는 수치는 분석 노트에도 없다는 뜻이므로 "확인 불가"로 남긴다 — 기준일을 지어내서 채우지 않는다.
- 인과 주장 레드라인: "정책 변경 때문에 대출 이자 부담이 커졌다"처럼 원인과 결과를 연결하는 서술도 숫자와 동일한 근거 기준을 따른다. 원문 기사 또는 분석 노트의 확인된 지표에서 근거를 댈 수 없는 구조적 인과 서술은 쓰지 않는다 — 못 대면 그 문장 자체를 빼거나 "확인 불가"로 남긴다.

## 개인화 조언 금지
"지금 사라/팔라", 특정 종목 매수·매도 권유는 절대 넣지 않는다. "이 업종은 항상 유리하다"류로 방향성 자산군 단정을 반복하는 것도 금지한다 — 같은 결론(예: "채권이 매력적이다")을 문단마다 다른 말로 되풀이하지 않는다. 일반적 맥락 설명에 그친다. 면책 문구는 본문에 쓰지 않는다 — 모든 글 하단에 `layouts/partials/disclaimer.html`을 통해 자동으로 표시된다.

## AI 흔적 자가검토
초안(및 용어사전 초안)을 다 쓴 뒤, 파일에 저장하기 전에 전체를 다시 읽으며 아래 일곱 가지를 점검하고, 걸리는 부분은 그 자리에서 고쳐 쓴다. 이 단계는 표현만 다듬는다 — 위 "숫자·사실관계 규칙"(지어내지 않기, 기준일 병기, 확인 불가 표시)은 그대로 유지하고 절대 손대지 않는다.

1. **예고형 서론**: "~에 대해 알아보겠습니다", "~를 짚어보겠습니다", "~가 화제입니다"처럼 예고만 하고 답을 미루는 문장이 남아 있는가. 있으면 삭제하고 결론으로 바로 시작한다.
2. **공허한 맺음말**: "이상으로 ~에 대해 알아보았습니다", "도움이 되셨길 바랍니다" 류의, 정보 가치가 0인 맺음이 있는가. 있으면 삭제하고, 마지막 문단은 핵심을 한 번 더 못박거나 "앞으로 지켜볼 지점"으로 끝낸다.
3. **추상적 형용사 남발**: "매우 중요한", "다양한", "효과적인" 같은 형용사가 반복되는가. 있으면 구체적 수치·사례로 바꾼다(수치를 새로 만들지 말고, 이미 본문에 있는 사실을 재활용한다).
4. **억지 나열**: 내용상 자연스럽지 않은데 "첫째/둘째/셋째"로 기계적으로 쪼갠 단락이 있는가. 있으면 자연스러운 산문으로 풀어 쓴다. (참고: draft.md §2의 "세 렌즈" 분류 자체는 분석 프레임워크이므로 유지하되, 그걸 본문에 옮길 때 기계적 나열체가 되지 않게 쓴다.)
5. **문장 리듬**: 문장 길이가 전부 비슷한가(예: 대부분 40~60자). 그렇다면 짧은 문장과 긴 문장을 섞어 리듬을 만든다.
6. **문체 통일**: 기존 발행글과 같은 존댓말체(~습니다/~입니다)로 통일되어 있는가. 뉴스 기사체 종결어미('~다/~했다/~이다')가 섞여 있으면 모두 '~습니다/~입니다'로 바꿔 쓴다.
7. **볼드체 잔존**: 본문(및 함께 쓴 용어사전 초안)에 `**`가 하나라도 남아 있는가. `grep -n '\*\*'`로 직접 확인하고, 있으면 전부 제거해 강조 없는 문장으로 고쳐 쓴다.

이 일곱 항목을 다 통과한 버전만 최종 파일 내용으로 확정한다.
