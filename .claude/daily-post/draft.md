# 작성 단계 지침 (daily-post 작성 단계)

## 1. 포스트 초안 작성
`content/posts/<slug>.md` 에 초안을 쓴다. slug는 영문 소문자-하이픈.

파일을 쓰기 전에 `content/posts/<slug>.md`가 이미 존재하는지 확인한다.
- **수동 모드**: 이미 존재하면 같은 주제를 다시 다루는 것인지, 슬러그가 겹친 것뿐인지 사용자에게 명확히 알리고, 다른 슬러그를 쓰거나 기존 파일을 덮어쓸지 사용자 확인을 받은 후에만 진행한다. 확인 없이 기존 파일을 덮어쓰지 않는다.
- **무인 모드**: 사용자에게 물을 수 없다. 슬러그 뒤에 `-2`, `-3`… 순번 접미사를 붙여 겹치지 않는 슬러그를 찾아 그대로 진행한다. 기존 파일은 절대 덮어쓰지 않는다.

front matter:

    ---
    title: "..."
    date: <현재시각, +09:00>
    description: "..."
    tags: [...]
    draft: true
    source_url: "<입력받은 원문 URL>"
    related_articles:            # 1.5단계 결과가 있을 때만. 0건은 오류가 아니라 정상 동작이다 — 이 필드 전체를 생략한다(빈 배열 금지).
      - title: "<기사 제목>"
        url: "<기사 URL>"
        source: "<언론사 표시명>"
    ---

1.5단계가 넘긴 순서(오래된 순)를 그대로 유지한다. 재정렬하지 않는다.

description 작성 규칙은 `.claude/daily-post/writing-styles.md`의 "description 작성 규칙"을 따른다.

본문 규칙:
- 문체·표현(볼드체 금지, 비유 활용, 어려운 용어 풀어쓰기, 두괄식, 인용블록 요약, 문장 리듬, 표 사용 기준 등)은 `.claude/daily-post/writing-styles.md`를 따른다. 이 규칙은 포스트 본문과 용어사전 설명(§3) 모두에 적용된다.
- 본문에 원문 기사 링크를 직접 쓰지 않는다 — front matter의 source_url을 바탕으로 `layouts/partials/source_link.html`이 자동으로 하이퍼링크를 표시한다.
- 1.5단계에서 수집한 연관 기사도 본문에 직접 링크하지 않는다 — front matter의 related_articles를 바탕으로 `layouts/partials/related_articles.html`이 하단에 자동으로 목록을 표시한다. 1.5단계가 넘긴 항목이 없으면 필드를 생략한다(빈 배열을 쓰지 않는다).
- 구조: (1) 무슨 일이 있었나 → (2) 왜 중요한가 → (3) 나에게 무슨 의미인가 → (4) 투자 관점에서 보면(§2).
- front matter의 source_url에는 입력으로 받은 원문 기사 URL을 반드시 그대로 기록한다.

## 2. 투자 연관성 (2단계 분석 노트 반영)
포스트 끝에 "투자 관점에서 보면" 단락을 넣는다. analysis.md 단계에서 만든 분석 노트(건드리는 렌즈·선행/동행 구분·확인된 수치·자산군별 함의)를 쉬운 말로 풀어 쓴다. 네 필드 전부 아래 불릿에서 각각 소비한다 — 하나라도 본문에서 빠지면 안 된다. 표현 방식(문체, 숫자·사실관계 레드라인, 개인화 조언 금지 등)은 `.claude/daily-post/writing-styles.md`를 따른다.

- **세 렌즈로 해석**: 분석 노트의 "건드리는 렌즈"를 짚는다 — 경기 흐름(성장이 살아나나 식나) / 돈의 흐름·유동성(금리·신용 여건) / 기업 체력(실적·밸류에이션).
- **선행 vs 동행 구분**: 분석 노트의 "선행 vs 동행"을 그대로 살려, 지금 당장의 영향인지 앞으로의 경고인지 나눠 설명한다. 타이밍을 뭉뚱그리지 않는다.
- **확인된 수치 반영**: 분석 노트의 "확인된 수치"를 본문에 명시적으로 옮긴다. 2개 이상이면 writing-styles.md의 표 규칙대로 표로, 1개면 문장으로 녹여 쓴다. "확인 불가"였다면 그 사실도 숨기지 않고 "확인 불가"로 쓴다.
- **자산군별 함의**: 분석 노트의 "자산군별 함의"(원가 부담·반사이익 등 구조적 논리)를 이번 뉴스가 어떤 업종·자산군에 유리 또는 불리하게 작용하는지 한두 문장으로 쉽게 풀어 쓴다. 특정 종목이 아니라 업종 단위로 서술한다. 이 단락 끝에는 반드시 "일반적인 맥락 설명일 뿐 개별 투자판단은 독자 몫"이라는 취지의 문장을 넣는다 — 생략 불가.

## 3. 용어 사전 (필요 시)
본문에 새 핵심 용어가 등장하고 `content/dictionary/_terms.yaml`에 아직 슬러그가 없다면:
1. `content/dictionary/<term-slug>.md` 에 설명 초안을 쓴다(`draft: true`, tags에 "용어사전", description에 그 용어를 한 문장으로 정의).
2. `content/dictionary/_terms.yaml`에 새 블록을 append한다:

   ```yaml
   <term-slug>:
     title: "<용어>"
     aliases: ["<실제 기사에 나올 법한 동의어1>", "<동의어2>"]
   ```

   aliases는 문법적 활용형이 아니라, 실제로 다른 기사에서 이 용어를 다르게 부를 만한 표현 2~3개로 제한한다.

## 4. 위키 링크
1. **용어**: `content/dictionary/_terms.yaml`을 읽어 slug/title/aliases 전체 목록을 얻는다. 본문에서 title 또는 aliases 중 하나와 일치하는 표현이 나오면 상대링크를 건다. 예: `[기준금리](/dictionary/base-rate/)`.
2. **과거 포스트**: `content/posts/`를 스캔해(파일명·front matter title·tags) 이번 뉴스와 주제가 겹치는 과거 글 후보를 찾는다. 관련 있으면 같은 방식으로 링크한다. 예: `[호르무즈·홍해 공급망 이슈](/posts/hormuz-red-sea-oil-supply-shock/)`.
3. `[[...]]` 문법은 쓰지 않는다.

## 5. AI 흔적 자가검토
§1~4로 초안(및 용어사전 초안)을 다 쓴 뒤, 파일에 저장하기 전에 `.claude/daily-post/writing-styles.md`의 "AI 흔적 자가검토" 절차(7항목 체크리스트)를 따라 전체를 점검하고, 걸리는 부분은 그 자리에서 고쳐 쓴다.
