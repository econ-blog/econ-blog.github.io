# 발행글 대조 진단

**생성일:** 2026-07-21
**코퍼스:** 네이버 블로그 경제글 53건 (2018-03 ~ 2022-10)
**발행글:** content/posts/ 4건

`feature-spec.yaml`의 `genre_invariant` / `violation_direction`을 사람이 정할 때 쓰는 참고 자료다.
루프가 이 값으로 자동 판단하지 않는다.

## [25, 75] 밖 특성 5개와 판정

| 특성 | 백분위 | 코퍼스 중앙 → 발행글 | 판정 | 근거 |
|---|---:|---|---|---|
| formal_ending_ratio | 99% | 0.783 → 1.00 | **장르** | writing-styles.md가 존댓말 통일을 의무화. 게다가 코퍼스를 존댓말 ≥60%로 필터해 뽑았으므로 순환이기도 하다. |
| abstract_adj_per_1k | 21% | 0.367 → 0 | **장르(규칙 성공)** | 자가검토 3번이 추상 형용사를 금지한 결과. 낮은 쪽은 위반이 아니다. |
| filler_per_1k | 87% | 0.819 → 1.887 | **장르** | 초과분 전량이 다만(6.9배)·한편(3.9배)·즉(2.1배). 논증형 산문의 논리 접속사다. 공허한 filler(결국·종합적으로·무엇보다·이처럼·사실상)는 발행글에서 전부 0. 어휘 목록이 두 범주를 섞고 있다. |
| para_len_cv | 23% | 0.664 → 0.525 | **경계** | 구조 bucket이라 스펙상 패치 대상 제외. 공백 정규화 수정 전에는 11%였고 수정 후 23%로 올라왔다 — 대부분 서식 잡음이었다. |
| para_opening_jaccard | 25% | 0.027 → 0.019 | **경계** | 경계선상. 절대값이 양쪽 다 0에 가까워 실질 의미 없음. |
| sentence_len_cv | 8% | 0.540 → 0.366 | **AI 흔적** | 양쪽 다 한국어 산문이고 문장 길이 분산을 강제하는 스타일 규칙이 없다. 발행글이 사람 글보다 확연히 평평하다. 공백 정규화 수정 전후로 코퍼스 중앙이 0.539 → 0.540으로 무변 — 서식 잡음이 원인이 아님이 확인됐다. |

## 결론

패치 대상 자격이 있는 특성은 **`sentence_len_cv` 하나뿐**이다.

## 특성 스펙에 반영할 것

```yaml
formal_ending_ratio:  { genre_invariant: false }   # 존댓말 의무화 + 코퍼스 필터 순환
abstract_adj_per_1k:  { genre_invariant: true, violation_direction: high }
filler_per_1k:        { genre_invariant: false }   # 분리 전까지 사용 불가
sentence_len_cv:      { genre_invariant: true,  violation_direction: low }
para_len_cv:          { genre_invariant: false }   # 구조 bucket
```

`filler_per_1k`는 두 특성으로 쪼개면 살릴 수 있다:
- `vague_filler_per_1k` (결국/종합적으로/무엇보다/이처럼/사실상) — AI 흔적 후보
- `logical_connective_per_1k` (다만/즉/한편/특히) — 장르 지표, 패치 대상 아님

## sentence_len_cv 상세

| | 값 |
|---|---|
| 코퍼스 중앙 | 0.540 |
| 코퍼스 IQR | [0.469, 0.635] |
| 발행글 평균 | 0.366 (백분위 8%) |
| 발행글 개별 | 0.446 / 0.398 / 0.404 / 0.217 |

문장 길이 **평균**은 정상이다 (코퍼스 55.2자 / 발행글 54.0자, 백분위 38%).
벗어난 것은 평균이 아니라 **분산**이다.

`writing-styles.md`는 이미 두 곳(문체·표현 규칙의 문장 리듬 항목, 자가검토 5번)에서
문장 리듬을 요구하면서 "40~60자"라는 구간을
명시한다. 평균은 그 구간 안에 정상적으로 들어와 있고 분산만 압축돼 있다 —
숫자를 제시한 것이 회피 기준이 아니라 수렴 목표로 작동했다는 가설과 정합한다.

검증되지 않은 가설이다. 반증 설계: 규칙에서 "40~60자"를 빼고 다음 초안의
`sentence_len_cv`를 재측정한다.

## 신뢰도 한계

- 발행글 n=4. 4건 평균의 백분위이므로 신뢰구간이 넓다.
- 코퍼스는 자체 필터(존댓말 ≥60%, ≥800자, 비-리스트형, 광고 제외)를 통과한 것만 담았다.
  필터가 셀수록 코퍼스가 발행글을 닮아가고, 기준선으로서의 독립성이 약해진다.
- 발행글도 LLM이 썼다. "장르 차이"와 "사람이 문제 삼지 않은 AI 흔적"을 구분하지 못한다.
- 코퍼스 연도 분포가 2022년(35건)에 쏠려 있다.
- 코퍼스 원문에 네이버 에디터의 zero-width space가 1499개 있었다. 정규화에서
  걷어내기 전에는 `para_len_cv` 백분위가 11%로 나왔고 수정 후 23%가 됐다.
  같은 종류의 미발견 서식 잡음이 남아 있을 수 있다.

## 신규 특성 발굴을 시도한 결과 (discover.py)

사전 지정 없이 log-odds(정보적 디리클레 사전)로 코퍼스 대비 과대표현 패턴을
캤다. 대상은 주제 중립 위치(문장 끝 4음절 / 문장 첫 어절 / 문단 첫 두 어절)와
어절 unigram. 주제어는 발행글 3/4건 이상 등장 조건으로 걸렀다.

결과: 유의 항목 사실상 0건. 유일한 통과는 "이번"(z=2.00, 경계값 1.96 턱걸이)
이고 뉴스 해설에서 자연스러운 어휘라 장르로 본다.

**방법이 실패한 것이 아니라 표본이 부족하다.** 발행글 4건 ≈ 6,400자로는
log-odds가 유의성에 도달하지 못한다. 발굴을 하려면 초안 30~50건 규모의
텍스트가 필요하다 — 그것이 루프의 생성 기계를 지어야 할 실제 이유다
(패치 검증이 아니라 발굴 표본 확보).
