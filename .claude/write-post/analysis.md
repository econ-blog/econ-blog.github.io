# 분석 단계 지침 (write-post 2단계)

이 단계는 초안 작성 전에 "투자 관점" 근거를 마련하는 단계다. 결과(분석 노트)는 파일로 저장하지 않고, 다음 단계(draft.md)에 바로 이어서 쓴다.

## 1. 세 렌즈로 분류
이번 뉴스가 다음 중 무엇을 건드리는지 정한다(복수 가능):
1. **경기 흐름**(성장이 살아나나 식나)
2. **돈의 흐름/유동성**(금리·신용 여건이 풀리나 조이나)
3. **기업 체력**(실적·밸류에이션)

## 2. 관련 챕터만 골라 매크로 참고 노트 확인
`macro-reference.md`(같은 디렉터리)를 Read로 읽는다. 뉴스 주제와 맞는 챕터 절만 골라 참고하고, 다른 챕터는 읽지 않는다. 이 파일 자체가 원문 대체 요약이므로 원문 PDF는 더 이상 참조하지 않는다.

macro-reference.md도 "이 책에 의하면" 식으로 직접 인용하지 않는다. 인과 논리만 가져와 분석에 녹여쓴다.

## 3. tudul.md와 PDF는 근거로만, 직접 인용 금지
`tudul.md`(저장소 루트)와 위 PDF 모두 "이 책에 의하면", "OO 프레임워크에 따르면" 식으로 직접 인용하지 않는다. 프레임과 논리만 가져와 분석에 녹여쓴다.

## 4. 핵심 지표 1~2개만 타깃 검색
이번 뉴스와 가장 직결된 지표 1~2개를 정해 WebSearch/WebFetch로 확인한다. tudul.md의 4모듈 전체 스냅샷을 재구축하지 않는다(그건 별도 매크로 분석 에이전트의 역할이지 이 블로그의 역할이 아니다).
- 검색 없이 지표 수치를 단정하지 않는다.
- 검색에 실패하면 "확인 불가"로 남긴다.
- "Fed가 다음 달 내린다"류 단정 예측 금지 — 시장 가격/확률로 확인된 범위에서만("CME FedWatch 기준 O% 확률로 인하 베팅 중" 등, 검색 후 그 값을 쓴다).

## 5. 분석 노트 정리
아래 형식으로 정리해 draft.md 단계로 넘긴다(파일 저장 안 함):
- **건드리는 렌즈**: [경기/유동성/기업체력 중 해당하는 것, 복수 가능]
- **선행 vs 동행**: [지금 당장의 영향인지, 앞으로의 경고 신호인지]
- **확인된 수치**: [지표명 - 값 - 출처 - 기준일] 목록, 없으면 "확인 불가"
- **자산군별 함의**(2단계에서 읽은 챕터 반영): [원가 부담/반사이익 등 구조적 논리]
